    base.py          - the shared contract:

    av2.py           - wraps proven lidar_depth.sparse_depth_map -> REFERENCE adapter.
    kitti.py         - KITTI raw: P_rect.R_rect.Tr_velo->cam projection, rectified
                       pinhole, intrinsics from P_rect. (image_02 / image_03)
    _nuscenes_core.py- canonical 4-step lidar->ego->global->ego->cam motion-comp
                       projection, shared by nuScenes + Lyft.
    nuscenes.py      - nuscenes-devkit adapter (NUSCENES_VERSION env, default v1.0-mini).
    lyft.py          - lyft_dataset_sdk adapter (nuScenes format; reuses the core).

  dataset.py refactor:
    - BenchmarkDepthDataset: ONE benchmark-agnostic dataset over any adapter. Does
      the resize/scale, builds intrinsics-free lidar_cam anchor + theta_gt, adds
      orig_wh. .build(benchmark, root, cams, stride, target_hw) factory.
    - AV2SensorDepthDataset kept as a thin AV2 wrapper; .index (log_dir,cam,ts) and
      discover_logs() preserved so eval/infer are untouched for AV2.

  CLIs generalized:
    - NEW src/visualize_benchmark.py: same LiDAR-depth overlay for ANY benchmark
      (cross-dataset companion to lidar_depth.py).
    - train.py: --benchmark (default av2), --cams (default = adapter's set), --stride;
      builds via BenchmarkDepthDataset.build.
    - eval.py: --benchmark/--split/--cams; vehicle targets now via adapter.vehicle_
      targets(ref) + orig_wh scaling (AV2 numbers unchanged; others report n=0).
