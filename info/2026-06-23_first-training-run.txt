MODEL (v1)  src/calib_depth/model.py
  image -> DAv2 -> relative depth; CalibEncoder(image) -> theta via
  LearnableIntrinsics (per-camera latent + image-conditioned delta). theta -> ray
  map conditions a small MetricHead([rel, cond, image]) -> metric depth-along-ray.
  Training res 518x518; intrinsics + lidar uv scaled to match.

BUGS FOUND & FIXED (the NaN saga)
  1. AMP fp16 overflow on metre-scale 3D losses -> NaN. Fix: train in fp32
     (model is tiny, <1 GB) + grad clip + skip non-finite loss/grad steps.
  2. ray_map FOV channel acos(z) has INFINITE gradient on the optical axis
     (ray=(0,0,1)). Fix: clamp acos input to [-1+1e-4, 1-1e-4].
  3. Freely-learned Brown-Conrady distortion makes undistort() diverge -> NaN.
     Fix: v1 learns only (log_fx, log_fy, cx, cy); distortion held at 0 (AV2 ring
     cams are near-pinhole). Re-enable later with strong constraints if needed.
  4. Launch bug: `pkill -f calib_depth.train` matched the SSH shell running it
     (pattern in its own cmdline) -> killed its own session / training. Fix: use
     `pgrep/pkill -f "[c]alib_depth.train"` bracket trick, or launch w/o pkill.
  5. opencv libGL ok on remote; empty AdamW param group (frozen backbone) -> drop
     empty groups.
